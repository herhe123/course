## 编程路线

### 第一步：计算机原理

想象一下你作算术题

CPU/内存：CPU相当于人的大脑，内存相当于草稿纸，注意，是草稿纸。你说CPU中还有缓存，那相当于心算，并不能保证记住所有的东西。

**计算机是由CPU、内存、以及CPU与内存之间的总线构成的，所有其他东西，什么键盘、显示器、鼠标、硬盘这些东西都是外围设备。**这个其实是最重要的知识。学习计算机原理最重要的就是要知道什么“是”计算机，什么“**不是“**计算机。因为后面的知识都是围绕这个基本原理进行设计的。例如显卡不是计算机的组成部分，所以CPU计算出来的显示信号实际上是要通过外围设备的“显卡”来提供显示功能的，而显卡的GPU是有自己独立的显存的，事实上显卡就是一个独立的特殊用途的计算机。有了这个基本理解，你就会明白现在ARM体系中的SOC，就是把GPU和CPU烧到一个芯片中共享内存，这是一个多么大的突破。

计算机内存是个线性的表，你可以理解为Excel里面的第一列。也就是说你给定行号，就定位了这个内存位置。一个内存位置对应了一个字节。也就是说你告诉内存我要读这个地址，内存就把这个地址的字节的内容返回给你；你要写一个字节到这个地址，内存就把你提供的字节内容写到这个地址。还有一些字长的内存地址转换，就不详细说了。

计算机原理学到这里就可以了。过程，半小时。

### 第二步：网络

千万不要去学计算机专业的网络教程。那门课是教你怎么写网络协议的入门课程，跟绝大多数程序猿对网络的使用是没有用的。

学CCNA培训教程。B站上应该有很多这样的课，我最喜欢的是一个油管上的英文课，但你可能上不了。B站上就行了，课程不好的话就稍微费点儿劲，总共也花不了多长时间，大概3天吧。

随着学习的深入，可能需要学CCNP的课程，那个后面再说吧，如果有时间在学到本课程表最后一步的时候可以看看，在这一步看的话知识量太大，容易看不懂。

为什么不建议学NP或者IE？网络类别的知识对于程序猿来说需要知道基本概念，你并不是去作网管，所以不需要更高级的知识，但如果对基本知识不了解，让你去AWS搭个网络环境你都不知道怎么搭。AWS/Azure/GCP/阿里云上建虚拟网络中所涉及的术语都看不懂，于是每一步都要查大量的文档，然后对出现的故障还不知所云。这是效率很低的。将来软件开发作到高级的话是一定要作软件架构的，软件部署架构对网络知识的依赖性是非常强的。企业级软件还有的一件事就是容灾，而容灾过程中涉及大量网络切换相关知识，你没有CCNA的知识的话迟早是要补课的。

### 第三步：虚拟机和操作系统

Linux管理员认证

操作系统原理，要理解进程和线程的概念。这方面可以B站油管找视频理解一下。除此之外操作系统原理这门课的内容是不需要学的。操作系统原理这门课讲的是操作系统分为哪些模块，每个模块怎么写。其中分哪些模块这个部分应该学，而具体细节部分不需要学。

操作系统分为哪些模块：参照计算机原理，计算机的组成部分是CPU、内存、总线以及外设构成的。操作系统的模块与之对应：1、管理CPU的叫处理机管理；2、管理内存的叫内存管理；3、管理外设的叫设备管理；4、外加一个计算机原理里面没有的，人机交互模块，也叫Shell，这里有图形化界面的Shell，和命令行界面的shell。好像漏了总线，总线是通过总线控制器管理，完全硬件实现，所以不需要管理模块。

半个小时讲完操作系统原理。进程和线程的概念讲了十几分钟，这里不确定娃学没学明白，无所谓了，以后再说。然后先搞个VMWare Player，找个视频看看怎么创建虚拟机。然后安装一个操作系统。

操作系统的选择最好是一个Linux的操作系统，B站或者油管上找个Linux系统管理员的课程。非常多。大概三天吧。

操作系统的技能训练对于程序猿来说很重要，但这个阶段没必要作那么多的训练，以后再说。

### 第四步：学习计算机语言

注意，是学习计算机语言，不是学习编程序。

HTML/JS/CSS。为什么选这个？因为直观。B站或油管上找个视频，大概2天最多了。

### 第五步：学习github

同样油管B站自己找课，大概两个小时。以后可以把你的练习程序都存到github上。

### 第六步：编程序

学习编程序和学习计算机语言之间最大的区别是，学习编程序的过程中主要学的是如何构建《数据结构》并基于数据结构构建算法。这个过程是计算机专业最重要的专业课。

学习方法：有人带的话基本上2个星期。没人带的话可能要一个月。同样B站或者油管上找课。因为这部分过于基础，所以课程有的是。但一般不用追求全部都学完，学好线性表、二叉树、图就可以了。如果你是看书学的话，可以找一本数据结构的教材。至于什么教材，没太大所谓，这种基础中的基础课，什么教材都行，随便找一本比较主流的大学数据结构教材就行了。

重要的的是编程训练。按照书学的，就把例题和课后作业都编一遍；如果在网上找课的，就自己在网上找题库，把线性表、二叉树、图的练习题都自己编一遍。编程语言就用上面第二步学的JS就可以。大多数书都会用Pascal 或者 C语言。没必要。

到这个时候，学习一门新的计算机语言一般不会超过2个小时。

### 第七步：写一个网站，同时学数据库

这里只学习关系型数据库，非关系型数据库以后再说。

关系型数据库是若干个二维表的集合。每个二维表就像一个Excel表单一样，每个列有个列名，然后就是一行一行的数据了。数据库操作的语言叫SQL。油管B站找个SQLLITE的课上一下，大概一个小时吧。

然后油管B站找个写网站的视频让它带着你写一个小网站，网站要用到HTML/JS/CSS，SQLLITE。有的课程可能是用Python做的，因为这些年Python比较火嘛。不过无所谓，到这一步，单独学个Python也就俩小时的事。

大多数这类小网站课程本身也就1个小时，自己整个写下来大概要2、3个小时吧。我带娃的时候用的是这个[GitHub - techwithtim/Flask-Web-App-Tutorial: Code for the note storing flask web app made during a YouTube video.](https://github.com/techwithtim/Flask-Web-App-Tutorial) 

到这里你应该学会什么叫前端、什么叫后端，一个典型B/S应用的基本架构。

### 第八步：部署

写完之后把你写的网站部署到第三步你创建的Linux虚拟机上。然后从你的宿主操作系统访问你虚拟机上你部署好的网站。这一步我没见过教程有涉及，可能会遇到一些困难，利用上面学到的CCNA的网络知识，利用各种论坛问问题，然后想办法解决。这一步往往需要花的时间比较多。找个有经验的人帮忙看看可以省不少时间，这一步实际上把前面学的网络和操作系统的知识串起来。

这一步的时间消耗不确定，有人带的话一两个小时吧，没人带自己摸索的话可能一个坎卡住你连问问题都不知道怎么问，要花几天的时间。上网找论坛问问题自己解决最好，练手啊！！

### 第九步：自动化部署——DevOps初步

把第七步写的网站保存到Github。在GitHub上创建一个release。写一个BASH脚本，能够自动化的下载这个release，并部署到前面你创建的Linux虚拟机上。这个过程需要：用Linux命令下载release包，可以是curl或者wget；解压缩包；使用nohup启动web应用以确保在你logout你的操作系统之后这个应用还能继续运行。

这一步需要学习BASH编程，一个小时吧；后面的部署过程就是自己找命令，测试命令，并把命令写进脚本的过程了整个时间对于新手得半天吧。

### 第十步：Github Action——DevOps自动化

B站油管找课学习Github Action，学习Github的branch概念和pull request的概念，然后作个CI/CD pipeline来激活并测试你自己写的网站。基本逻辑就是使用github runner（一个Linux虚拟机），在pull request的事件驱动下启动runner，启动你的网站，并测试你的网站端口是不是处于侦听状态（这是比较简单的验证逻辑，仅仅用于学习）。能力强的可以写一个状态API来检测你的网站是不是起来了。

这个过程对于新手可能要一天。

现在的软件开发都已经DevOps自动化了。这个东西要是不懂就没法融入软件开发。

### 第十一步：PKI

找一些PKI的介绍性文章理解PKI是什么东西，尤其理解SSL/TLS是怎么样建立信任关系的。理解CA/Certificate之间的关系。自己创建一个自签名的CA证书。

这个过程也就几个小时吧。

### 第十二步：AD

Active Directory

目前大部分的用户/授权管理都是通过目录服务完成的。所以理解目录服务是挺重要的。找一个**Active Directory**的培训视频看一看，大概3天吧，应该能对这个有比较充分的了解了。可能记不住，无所谓，有个基本概念的印象最重要。

之所以学AD是因为目录服务类课程中AD的免费教程最好找，也最直观。好多人觉得AD是安全范畴，而实际上是目前软件开发中所涉及到的用户管理，用户验证，联邦用户管理，授权管理等等概念都是通过目录服务完成的，没有目录服务的知识的话，迟早要补课。现在也没有哪个企业的服务器没有目录服务管理着。所以这真的是必修课。

### 第十三步：云计算初步

上AWS或者Azure或者GCP或者阿里云，任何一个你能访问到的，推荐AWS。部署一个网络和Linux虚拟机，用第八步写的脚本把应用部署到这个虚拟机上，并从internet访问这个应用。

这个步骤在油管B站上有无数的视频。几个小时吧。

### 第十四步：系统设计

前面十几个步骤学下来基本上可以说软件开发入门了。下面进入系统设计。大多数初级中级高级程序猿的面试都会出现大量的系统设计的问题，而系统设计基本没有课程可以讲，因为这纯粹是个实践内容。最好的学习方法是在油管B站找一些简单的系统设计介绍，稍微看看，然后去考AWS或Azure的Solution Architect的认证。这个准备过程可能要两三个月。就是去买个题库，一道一道题的刷，遇到有不会的问题去网上问。我用的是ExamTopics，这个网站的好处是每道题下面面都有人的讨论，绝大多数题你可以知道正确答案为什么对，错误答案为什么错；坏处是网站本身提供的答案有大量的错误，不过讨论区给出的答案基本上是靠谱的。

整个学习过程中每道题涉及到的不懂的术语和知识点都要弄明白。这里说的弄明白不是说每个细节都要明白，而是要知道这个术语是什么意思，它所涉及的功能是什么，能满足什么样的需求。这个过程中需要针对各种概念去网上找答案，过程挺烦的。以后有时间我再整理这方面的基础知识。

这个过程我带娃学了8个月。一共600道题左右，开始的时候知识点不熟悉，每周作10道题，后来每周作30-40道题左右。娃还有自己学校的学习任务不能全身心的铺在这个上面。8个月拿下了AWS Solution Architect Associate的认证。

不能说认证完了就行了，但认证完了绝对够资格成为一个入门级程序猿或者说IT工程师了。

# 前言

其实所有课程的学习过程中，绪论最重要，其次是每个章节的第一个小节。没有人能够上来就记住所有细节。好的记忆方式从来都是先有条理的知道知识的分类，按照分类去记忆知识点。所以不要小看绪论，好好学。

# 计算机原理

***现代计算机是由CPU和内存以及连接CPU和内存的总线构成的***。这句话的重点是：计算机的组成是不包括硬盘、网卡、显卡、声卡、显示器、键盘这些东西的。计算机仅仅是由CPU、内存和总线构成的。我们看到硬盘、网卡、声卡、显卡、USB接口这些东西叫做***外围设备***，简称***外设***。而显示器、键盘、鼠标是更外层的东西，计算机并不和这些设备打交道。虽然计算机本身不包括外设，计算机计算的结果要通过外设实现持续性（存储）以及与人的交互，所以外设很重要。

到此，对于非计算机专业的人就可以了。对于计算机专业学生，按照CPU、内存、外设去学习计算机原理。

# 操作系统

操作系统的目的是虚拟化计算机资源给多个用户使用，这里的用户不一定是人，例如你一边用电脑放音乐一边用这个电脑打字。如果不需要虚拟化计算机资源给多个任务使用，那么就不用那么复杂的操作系统了。

所以操作系统需要虚拟化的资源有：CPU、内存、外设。所以操作系统就分为处理机管理、内存管理、外设管理。

- 处理机管理是通过进程管理和线程管理实现的。需要理解进程和线程是什么东西。
- 内存管理就是在管理一个大的线性表的分配与回收，虚拟内存的映射和抽象。参照《数据结构》理解什么是线性表。
- 外设管理实际上就是管理计算机与外设的通信。
- 文件系统属于外设管理的一部分，但因为非常非常的重要，多数教材会把这个单独拿出来讲。

**对于非科班读者，到此《操作系统》这门课基本就不用再深入了解了。了解了也没用。更好的方式是去学一个Linux 管理员培训，B站上有的是相关资源。**

没有哪个程序猿自己不会搭软件运行环境，如果有的话那就一定是个不合格的程序猿。软件的安装、卸载、配置，版本同步，服务的启动、停止、配置，存储的管理，系统参数调整，时间同步，所有这些东西可以不用系统化的学习，我就没系统化的学过，但必须要知道。我还是觉得系统化的学一下比较好，打个基础。

# 数据结构

数据结构讲的内容就是线性表、树、图这三种数据结构及其子数据结构和围绕着这些数据结构的基础算法。

其中数据结构有

线性表：数组、链表；线性表的应用：队列、栈；

树：树、二叉树、B树；

图：图

基础算法有：遍历、查找、插入、排序、递归。就是针对上面每一种数据结构怎么实现这些基础算法。例如给你一个线性表，两种不同的实现方式，一种是数组、一种是连表，怎么实现查找、插入、排序。换成二叉树，两种不同的实现方式，一种是数组、一种是连表，怎么实现，查找、插入、排序、递归。

**数据结构的学习是最难的。计算机原理和操作系统这两门课对于非专业人士来说学到上面提过的内容就行了。而数据结构这门课需要踏踏实实的去学所有的内容，把每一个章节的课后习题都用程序实现一遍。**

# 总结

1. 《数据结构》：这个是软件开发的必修，只要是干软件的，这个一定要学。
2. 一门面向结构的开发语言：建议基础语言C，一般学数据结构的同时就学完了
3. 一门面向对象的开发语言：建议Java，面向对象的概念和面向结构的概念是不同的
4. 一门应用服务器中间件，加上任何一个数据库Oracle、MY SQL、SQL Server、DB2是值得学习的。
5. HTML/CSS/JavaScript
6. CCNA，Windows Active Directory，一门Linux OS Admin虽然并不是软件开发的必备技能，但在长期成长中是必要技能，应该有学习。对于想搞IT部门工作的，这个部分是非常推荐的。
7. 一门脚本语言：Java Script, BASH, Perl, Python, Powershell。有上面两个语言的基础，这个基本靠google或百度就行了，不需要专门去学。

在现在的体系下，按照上面顺序从1学到4然后开始找软件开发的工作。想搞前端的学到5，再出去找工作。也可以单学6然后找IT部门的工作。这些部分基本上是一个好的IT技术工作者的基本功。有精力的话最好都学一下。

## 其他方面学习选择

如果你有课余精力的话，尽可能的去实践，而不是学另外一个没用的东西。

英语之外的语言类专业除非你决定将来从事那个语言的外包类工作，否则没有任何意义。

英语对长期的技术成长很重要，但真的还不如加入一个美剧的字幕组靠谱。作为20年的IT从业者，我要是面试官的话会很看重你字幕组的工作经验而忽略你辅修英语专业的文凭。你告诉我哪个剧是你翻的，并举一些例子证明这部剧是你翻的，并说一些真实例子证明你遇到的困难和有趣的事情，我差不多能选择立马招你。面对一个本科毕业生，我根本不在乎你干的是什么，我在乎的是1. 你对IT有基本的了解；2. 你的工作态度和工作能力。

工商管理专业+计算机专业是进入大多数咨询公司的良好条件。非常多的咨询公司作ERP、Finance、HR、SCM这些实施，但招不到同时懂工商管理和计算机的人才。

金融有点虚，对找工作实际上作用不大，可能会有一些比较窄的领域会对金融+计算机感兴趣。
